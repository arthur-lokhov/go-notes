# 📛 Константы в Go

Представь, что ты пишешь рецепт торта. Ты знаешь, что нужно ровно 200 грамм сахара. Не «примерно 200», не «200, но можно поменять». Ровно 200.

**Константа в Go** — это такое же фиксированное значение. Оно известно на этапе написания рецепта (компиляции) и не может измениться, когда торт печётся (выполнение программы).

Но константы в Go — это не просто «неизменяемые переменные». Это мощный инструмент, который даёт тебе производительность, безопасность и гибкость, недоступную в других языках.

---

## 🎯 Зачем нужны константы

**Первая причина:** производительность.

Когда компилятор видит константу, он **заменяет её значением прямо в коде**. Этот процесс называется **инлайнинг**.

```go
const Pi = 3.14159

func CircleArea(r float64) float64 {
    return Pi * r * r
}
```

В скомпилированном коде не будет обращения к памяти за значением `Pi`. Компилятор подставит `3.14159` прямо в инструкцию умножения. Это быстрее, чем читать переменную из памяти.

**Вторая причина:** безопасность.

Ты не можешь случайно изменить константу. Компилятор не позволит.

```go
const MaxUsers = 100
MaxUsers = 200  // ❌ Ошибка компиляции: cannot assign to MaxUsers
```

**Третья причина:** читаемость.

```go
// ❌ Плохо
if status == 1 {
    // ...
}

// ✅ Лучше
const StatusActive = 1
if status == StatusActive {
    // ...
}
```

Второй вариант говорит **что** ты проверяешь, а не **как** это представлено в памяти.

## 📝 Как объявлять константы

### Базовый синтаксис

```go
const Name = value
```

Всё просто. Но есть нюансы.

**Константа должна быть известна на этапе компиляции.** Ты не можешь вызвать функцию:

```go
const Now = time.Now()  // ❌ Ошибка: time.Now() не константное выражение
```

**Но можно использовать константные выражения:**

```go
const A = 100
const B = len("hello")  // ✅ len() — встроенная функция, известна компилятору
const C = A + B         // ✅ 100 + 5 = 105
```

### Групповое объявление

```go
const (
    StatusPending = 0
    StatusActive  = 1
    StatusInactive = 2
)
```

Это удобнее, чем писать `const` для каждой константы. И именно здесь появляется `iota`.

## 🔢 `iota`: магический счётчик

`iota` — это встроенный идентификатор, который **автоматически увеличивается** в блоке `const`.

```go
const (
    A = iota  // 0
    B         // 1
    C         // 2
)
```

**Как это работает:**

- `iota` начинается с 0 в каждом блоке `const`
- На каждой новой строке `iota` увеличивается на 1
- Если нет явного присваивания, выражение повторяется с предыдущей строки

**Пример:**

```go
const (
    A = iota  // iota = 0, A = 0
    B         // iota = 1, B = 1 (повторяет A = iota)
    C         // iota = 2, C = 2
)
```

### Пропуск значений с `_`

Пустой идентификатор `_` отбрасывает значение, но **не останавливает `iota`**:

```go
const (
    _ = iota  // iota = 0, значение отброшено
    A         // iota = 1, A = 1
    B         // iota = 2, B = 2
)
```

### Битовые маски с `iota`

`iota` особенно удобен для битовых флагов:

```go
const (
    _ = iota                   // 0 (пропускаем)
    KB = 1 << (10 * iota)      // 1 << 10 = 1024
    MB                         // 1 << 20 = 1048576
    GB                         // 1 << 30 = 1073741824
)
```

Здесь каждая константа — степень двойки. `iota` увеличивается, выражение `1 << (10 * iota)` повторяется.

**Другой пример — флаги доступа:**

```go
const (
    FlagRead  = 1 << iota  // 1 (0001)
    FlagWrite              // 2 (0010)
    FlagExec               // 4 (0100)
    FlagAll = FlagRead | FlagWrite | FlagExec  // 7 (0111)
)
```

Теперь можно комбинировать флаги:

```go
perms := FlagRead | FlagWrite  // 3 (0011)
```

### Подводные камни `iota`

**`iota` привязан к строке, а не к значению.**

```go
const (
    A = iota + 1  // iota = 0, A = 1
    B             // iota = 1, B = 2 (повторяет iota + 1)
    C = iota      // iota = 2, C = 2 (использует текущее iota)
)
// A=1, B=2, C=2
```

Здесь `C` равно 2, потому что находится на третьей строке (индекс 2), а не потому что `B` было равно 2.

**Сброс выражения:**

```go
const (
    A = iota  // iota = 0, A = 0
    B         // iota = 1, B = 1
    C = 100   // iota = 2, C = 100 (выражение сброшено)
    D         // iota = 3, D = 100 (повторяет C = 100)
    E = iota  // iota = 4, E = 4 (явно указан iota)
)
```

## 🎭 Типизация констант

В Go константы бывают **типизированные** и **нетипизированные**. Это важное различие.

### Типизированные константы

```go
const x int = 10
```

Здесь `x` имеет тип `int`. Она ведёт себя как переменная типа `int`, только неизменяемая.

### Нетипизированные константы

```go
const y = 20
```

У `y` нет явного типа. Компилятор выводит тип из контекста, где константа используется.

**Почему это важно?** Нетипизированные константы более гибкие:

```go
const z = 20

var a int = z    // ✅ z становится int
var b int64 = z  // ✅ z становится int64
var c float64 = z // ✅ z становится float64
```

Одна и та же константа может быть использована как `int`, `int64`, `float64` — в зависимости от контекста.

### Арифметика произвольной точности

Нетипизированные константы поддерживают **вычисления без потери точности** на этапе компиляции.

```go
const Huge = 1e1000  // Число, которое не помещается ни в один тип
const Small = 1.0 / Huge  // +0.000000e+000

const AnotherHuge = Huge / 1e999  // 10.0
var f float64 = AnotherHuge  // ✅ 10.0 помещается в float64
```

Здесь `Huge` слишком велико для `float64`, но компилятор позволяет работать с ним, пока результат помещается в тип.

**Но:**

```go
const Huge = 1e1000
var f float64 = Huge  // ❌ Ошибка: constant 1e+1000 overflows float64
```

Когда ты присваиваешь константу переменной, компилятор проверяет, помещается ли значение в тип.

## ⚠️ Ограничения констант

**Нельзя:**

- Вызывать функции
- Брать адрес (`&`)
- Использовать слайсы, мапы, структуры

**Почему нельзя взять адрес?**

Константы существуют только на этапе компиляции. В скомпилированном коде их нет — они заменены значениями.

```go
const X = 5
var p = &X  // ❌ Ошибка: cannot take the address of X
```

У `X` нет адреса в памяти — его просто не существует во время выполнения.

## 🔄 Константы и интерфейсы

Когда нетипизированная константа присваивается `interface{}`, она получает **конкретный тип по умолчанию**:

- Целые числа → `int`
- Числа с плавающей точкой → `float64`
- Руны → `rune`
- Строки → `string`

```go
const X = 5
var i interface{} = X  // i хранит int(5)

switch i.(type) {
case int:
    fmt.Println("Это int!")  // ✅ Сработает этот вариант
}
```

Интерфейс не хранит «untyped» значение — он хранит значение с конкретным типом.

## 🏛️ Паттерн `enum`

В Go нет встроенных перечислений, как в Java или C#. Но их можно эмулировать с помощью констант и пользовательских типов.

```go
type Weekday int

const (
    Sunday Weekday = iota
    Monday
    Tuesday
    Wednesday
    Thursday
    Friday
    Saturday
)
```

Здесь `Sunday` — не просто `0`, а значение типа `Weekday`. Это предотвращает случайное смешивание:

```go
func IsWeekend(day Weekday) bool {
    return day == Sunday || day == Saturday
}

IsWeekend(0)        // ❌ Ошибка типа: 0 — int, не Weekday
IsWeekend(Sunday)   // ✅ Правильно
```

### Проблема: отладка

Числовые `enum` сложно отлаживать. Когда ты видишь `0` в логе, ты не знаешь, это `Sunday` или `Monday`.

**Решение:** генератор `stringer`.

1. Установи:

```bash
go install golang.org/x/tools/cmd/stringer@latest
```

2. Добавь комментарий:

```go
//go:generate stringer -type=Weekday
type Weekday int
```

3. Запусти:

```bash
go generate ./...
```

Будет создан файл `weekday_string.go` с методом `String()`:

```go
fmt.Println(Sunday)  // Выведет: Sunday, а не 0
```

### Сериализация в JSON

По умолчанию `enum` сериализуются в числа:

```go
data, _ := json.Marshal(Sunday)
fmt.Println(string(data))  // Выведет: 0
```

**Чтобы получить строку:**

```go
func (w Weekday) MarshalJSON() ([]byte, error) {
    return json.Marshal(w.String())
}
```

Теперь:

```go
data, _ := json.Marshal(Sunday)
fmt.Println(string(data))  // Выведет: "Sunday"
```

## 🧬 Внутреннее устройство

### constant.Value

Внутри компилятора константы представлены типом `constant.Value` из пакета `go/constant`. Этот тип может хранить:

- Числа произвольной точности
- Строки
- Булевы значения
- Специальное значение `unknown`

### Constant Folding

Компилятор выполняет **свёртку констант** — вычисляет константные выражения на этапе компиляции.

```go
const C = 2 * 5  // Компилятор заменит на 10
```

Это происходит на этапе SSA (о нём в главе про internals). Компилятор видит выражение, вычисляет результат, подставляет вместо выражения.

### types.Const

В пакете `go/types` константа представлена структурой:

```go
type Const struct {
    obj
    val constant.Value  // Точное значение константы
}
```

`val` хранит значение с произвольной точностью. Это позволяет тайп-чекеру работать с константами без потери точности.

### Исключения из инлайнинга

Обычно константы инлайнятся. Но есть исключения:

- **Экспортируемые константы:** их значения записываются в `.a` файл, чтобы другие пакеты могли использовать.
- **Использование с `reflect`:** когда константа передаётся в `reflect.ValueOf()`, её значение должно быть доступно во время выполнения.

## 🏆 Лучшие практики

**1. Избегай магических чисел**

```go
// ❌ Плохо
if status == 1 {
    // ...
}

// ✅ Лучше
const StatusActive = 1
if status == StatusActive {
    // ...
}
```

**2. Для `enum` задавай явный тип**

```go
type Status int  // Явный тип

const (
    StatusPending Status = iota
    StatusActive
    StatusInactive
)
```

Это предотвращает смешивание разных перечислений.

**3. Не смешивай семантику в одном блоке `iota`**

```go
// ❌ Плохо
const (
    StatusActive = iota  // Статус
    RoleAdmin            // Роль
)

// ✅ Лучше
const (
    StatusActive = iota
    StatusInactive
)

const (
    RoleUser = iota
    RoleAdmin
)
```

**4. Строковые vs числовые константы**

| Тип | Плюсы | Минусы |
|-----|-------|--------|
| **Числовые** | Эффективны, можно как битовые флаги | При отладке `0` менее информативен |
| **Строковые** | Читаемы, хороши для JSON | Менее производительны, возможны опечатки |

**Рекомендация:** используй `iota` для внутренних состояний, строковые константы для внешних API.

---

## 🎯 Итог

Константы в Go — это не «неизменяемые переменные». Это значения, известные на этапе компиляции, которые компилятор подставляет прямо в код.

**Запомни:**

- Константы инлайнятся — нет накладных расходов на чтение
- Нетипизированные константы гибче — тип выводится из контекста
- `iota` удобен для `enum` и битовых масок
- Нельзя взять адрес константы — её нет в памяти во время выполнения
- Для отладки `enum` используй `stringer`

Понимание констант поможет тебе писать более эффективный и читаемый код.
