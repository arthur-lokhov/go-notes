    🔧 Переменные в Go

    Переменная - это именованная область памяти, хранящая значение определенного типа. Тип переменной определяет множество допустимых значений и операций, которые можно над ней выполнять.

    Go предлагает строгую, но лаконичную систему работы с переменными, которая включает:

    • статическую типизацию;
    • автоматическую инициализацию (нулевыми значениями);
    • явное и неявное определение;
    • поддержку краткого синтаксиса :=;
    • различие между значением и ссылкой;
    • полную прозрачность в управлении памятью.

    ⚙️ Внутреннее устройство переменной

    Когда в Go объявляется переменная, она резервирует область памяти, соответствующую типу значения, и связывает её с именем. В зависимости от типа переменная:

    • Может быть значением (value type) - хранящим данные напрямую;
    • Может быть ссылкой (reference type) - указывающей на данные в куче.

    В примитивных типах данные хранятся в стеке, а ссылочные типы указывают на выделенную память в куче.

    Все переменные инициализируются нулевыми значениями (zero values).
    Даже если вы не задали значение явно.

    📌 Объявление переменных

    🧱 Классическое объявление

    Ключевое слово var используется для объявления переменной.

    var name Type = value

    Если значение указано, тип может быть выведен автоматически:

    var count = 42      // int
var enabled = true  // bool

    А если же значение не задано - тип указывать обязательно:

    var port int

    🧮 Множественное объявление

    Можно объявлять несколько переменных одной строкой:

    var x, y, z int

    Все переменные должны иметь один и тот же тип

    При множественном объявлении с явным указанием типа у всех переменных должен быть один и тот же тип.

    var a, b, c int // ✅ корректно: все int
var x, y string // ✅ корректно: все string
var d int, e string // ❌ ошибка компиляции

    ⚡️ Краткое объявление (:=)

    Краткое объявление нельзя использовать на уровне пакета - только внутри функций.

    В Go можно использовать краткую форму объявления:

    message := "hello"    // string
answer := 42          // int

    Эквивалент коду ниже:

    var message = "hello"
var answer = 42

    При кратком объявлении через := важно создавать хотя бы одну новую переменную, чтобы избежать ошибки.

    package main

import (
    "errors"
    "fmt"
)

func main() {
    x, err := 20, errors.New("some error") // Обе переменные новые

    fmt.Println(x, err)

    z, err := 30, errors.New("some error 2") // Лишь одна переменная новая, но это работает

    fmt.Println(z, err)

    x, err := 40, errors.New("some error 3") // ❌ Ошибка: нету новых переменных
}

    🌀 Статическая и динамическая типизация

    Каждая переменная в Go имеет статический тип, известный на этапе компиляции. Это означает, что нельзя присвоить переменной значение несовместимого типа:

    var x int // x имеет статический тип int
x = 42 // ✅
x = 4.2 // ❌ Ошибка: нельзя использовать 4.2 как значение int

    Но переменные интерфейсного типа (interface{}) также обладают динамическим типом - фактическим типом хранимого значения во время выполнения.

    var x interface{} // x имеет статический тип interface{}, нулевое значение
x = 42 // x теперь содержит значение 42, динамический тип - int
x = "hello" // x теперь содержит строку, динамический тип - string
    Почему это так работает

    В Go интерфейсная переменная (interface{}) устроена не так, как простые переменные.

    Она логически состоит из двух частей:

    • Type: динамический тип значения(например, int, string, *MyStruct).
    • Value: само значение этого типа.

    Упрощенно структуру интерфейса можно представить следующим образом:

    type iface struct {
    typ *itab           // Указатель на описание типа
    data unsafe.Pointer // Указатель на значение
}

    Важно понимать, что несмотря на то что interface{} может хранить значение любого типа, он делает это не сам по себе, а через внутреннюю структуру itab.

    Когда ты пишешь x = 42, Go сохраняет:

    • typ -> указатель на int тип;
    • data -> значение 42, приведенное к указателю.

    Когда ти пишешь x = "hello", предыдущие данные теряются (сборщик мусора утилизирует, если нет других ссылок на эти данные), и переменная теперь указывает на:

    • typ -> string;
    • data -> "hello".

    Про интерфейсы подробнее можно прочитать в соответствующей главе.

    🔁 Области видимости переменных

    В Go каждая переменная имеет область видимости — часть кода, где она доступна. Область видимости может быть:

    • Глобальной (пакетной) - переменная доступна во всем пакете;
    • Локальная - внутри функции, блока, цикла или условия.
    package main

import "fmt"

var a int = 1 // Переменная доступна глобально

func main() {
    var b int = 2 // Переменная доступна только внутри main()
    {
        var c int = 3 // Переменная доступна только внутри этого блока
        fmt.Println(a, b, c) // 1 2 3
    }
    fmt.Println(a, b) // 1 2
    fmt.Println(c) // ❌ Ошибка: переменная `c` не найдена
}

    🌐 Глобальные переменные

    Переменные, объявленные на уровне пакета (вне функций)^

    • Видимы во всех файлах пакета;
    • Инициализируются до вызова main();
    • Могут использоваться в init() функциях.
    var globalVar = "I'm available everywhere in package"

func init() {
    fmt.Println(globarVar) // Доступно
}

    👻 Shadowing (Сокрытие переменных)

    В Go можно переопределить переменную в локальной области видимости, даже если переменная с таким именем уже существует.

    x := 5
{
    x := 10
    fmt.Println(x) // 10
}
fmt.Println(x) // 5

    Shadowing может привести к путанице

    Для изменения значения существующей переменной используйте оператор присваивания =, а не :=, потому что иначе создается новая переменная и взаимодействие происходит с ней.

    x := 5
{
    x = 10
    fmt.Println(x) // 10
}
fmt.Println(x) // 10

    🔄 Обмен значениями (Swap)

    В Go можно легко поменять значения двух переменных местами.

    a, b := 1, 2
a, b = b, a
fmt.Println(a, b) // 2 1

    🔍 Игнорируемое имя переменной (_)

    В Go доступен специальный идентификатор _, называемый blank identifier (пустой идентификатор). Он используется, когда нужно получить, но не сохранять.

    Попытка чтения из _

    Если попробовать прочитать значение из _ - будет ошибка компиляции, о том что _ нельзя использовать как тип или значение.

    В контексте объявления переменных var и := он позволяет отбросить значение, не создавая именованную переменную.

    Если функция возвращает два значения, но тебе нужно только одно — можно использовать _.

    value, _ := someFunc() // получаем только первый результат
var _, err = someFunc() // получаем только второй результат

    Игнорирование всех значений

    Если попытаться проигнорировать все значения, например:

    package main

func foo() (int, int) {
    return 1, 2
}

func main() {
    _, _ := foo() 
}

    Программа не скомпилируется с ошибкой, которая сообщит нам, что с левой стороны не происходит считывание ни одного значения.

    🔨 new: выделение памяти под значение

    Функция new выделяет память под значение указанного типа и возвращает указатель на нулевое значение этого типа.

    p := new(int)
fmt.Println(*p) // 0
*p = 34
fmt.Println(*p) // 34

    Функция new применима к любому типу (и к примитивным, и к ссылочным).

    x := new(int)               // *int, содержит 0
s := new([]int)             // *[]int, содержит nil slice
m := new(map[int]string)    // *map[int]string, содержит nil map

    Функция new не инициализирует внутреннюю структуру типов

    Важно знать, что new(slice|map|chan) только создает указатель на nil, не инициализируя внутреннюю структуру.

    Чтобы получить рабочий slice|map|chan, нужно использовать make.

    🧰 make: инициализация ссылочных типов

    Функция make работает только со срезами, картами и каналами.

    Функция make - это встроенная функция компилятора, жестко органиченная на работу только с тремя встроенными типами.

    s := make([]int, 10)
m := make(map[string]int)
c := make(chan int)

    make возвращает инициализированный объект, а не указатель в отличии от new.

    package main

func main() {
    s1 := new([]int)
    fmt.Println(s1, *s1)    // Ссылка: &[], Значение []
    s2 := make([]int, 5)
    fmt.Println(s2)         // Значение: [0 0 0 0 0]
}

    Пока объект не проинициализирован в него нельзя писать данные.

    Не представляется возможным поместить в slice|map|chan значение пока они не проинициализированы.

    Исключением является append для срезов, он безопасен при работе с nil срезом.

    var s []int // nil int slice
s = append(s, 10) // работает
fmt.Println(s) // [10]

s[0] = 5 // ❌Ошибка: значение за пределами массива

    📛 Константы

    Константы (const) - это неизменяемые значения, определяемые во время компиляции:

    const Pi = 3.1415
const Greeting = "Hello"

    Тип можно указать явно или вывести автоматически.

    const x int = 10
const y = 20

    🔢 iota — автоинкремент для констант

    iota — это встроенный идентификатор, используемый в блоках const для генерации последовательностей.

    const (
    A = iota    // 0
    B           // 1
    C           // 2
)

    📌 Общая модель

    • iota - счетчик, начинающийся с 0 в каждом блоке const;
    • На каждой новой строке внутри блока iota увеличивается на 1;
    • Каждая строка - это независимая вычисляемая инструкция, даже если она "визуально" ничего не содержит (повторение предыдущего выражения).

    ⚙️ Использование _ в блоках с iota

    _ не влияет на iota: счетчик продолжает увеличиваться, даже если значение не используется.

    const (
    _ = iota    // iota = 0, значение отброшено
    A           // iota = 1, A = 1
    B           // iota = 2, B = 2
)

    Если использовать _ в середине блока const, то надо будет снова затриггерить iota.

    const (
    A = iota    // iota = 0, A = 0
    _           // 1, значение отброшено
    B = iota    // iota = 2, B = 2
    C           // iota = 3, C = 3
)

    🔄 Поведение при комбинированных выражениях

    Когда ты пишешь несколько выражений на одной строке, iota будет иметь одно и то же значение во всей этой строке:

    const (
    A, B = iota, iota   // 0, 0
    C, D = iota, iota   // 1, 1
)

    В последующих строках необязательно указывать iota

    const (
    A, B = iota, iota   // 0, 0
    C, D                // 1, 1
    E, F                // 2, 2
)

    Количество констант и инициализаторов iota должно совпадать.

    const (
    A, B    = iota, iota // 0, 0
    C, D                 // 1, 1
    E, F, G              // ❌ Ошибка: нету значения для инициализации G
)

    ⚠️ Перезапись и комбинирование значений

    Можно свободно смешивать выражения с iota и без - но iota продолжает увеличиваться по строкам, а не значимости выражений.

    const (
    A = iota            // iota = 0, A = 0
    B                   // iota = 1, B = 1 (повторяет значение A)
    C = 100             // iota = 2, C = 100
    D                   // iota = 3, D = 100 (повторяет значение C)
    E = iota * 2        // iota = 4, E = 8
    F, G = iota, iota   // iota = 5, F = 5, G = 5
    H = "static"        // iota = 6, H = "static"
    I                   // iota = 7, I = "static" (повторяет значение H)
)

    🛠 Использование iota в битовых масках

    iota можно использовать вместе с побитовыми сдвигами.

    Например, для расчета памяти:

    const (
    _ = iota                // 0 (пропускаем)
    KB = 1 << (10 * iota)   // 1 << 10 = 1024
    MB                      // 1 << 20 = 1048576
    GB                      // 1 << 30 = 1073741824
)

    Или для управления флагами:

    const (
    FlagNone = 0
    FlagRead = 1 << iota // 1 << 0 = 1
    FlagWrite            // 1 << 1 = 2
    FlagExec             // 1 << 2 = 4
    FlagAll  = FlagRead | FlagWrite | FlagExec // 7
)
